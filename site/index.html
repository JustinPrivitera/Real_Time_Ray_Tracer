Real-time Ray Tracing

Justin Privitera, Noah Stapp, Kevin Kellar - CSC 572, Fall 2020 - Christian Eckhardt

Project Description

For our CSC 572 final project, we created a real-time ray tracer using OpenGL compute shaders. Ray tracing is a rendering technique that uses simulated light rays and their bouncing to realistically render scenes. Done in real-time, ray tracing produces realistic-looking scenes at the cost of framerate.

Technical topics used include:

  • Compute shaders
  • Post-processing
  • Keyboard input
  • Texture generation + rendering
  • Lightning techniques/modes

Screenshot of finished project.

Key features

Our ray tracer implements several commonly-used rendering techniques to improve scene quality. The first, ray traced ambient occlusion, calculates how exposed each pixel in a scene is to the ambient lighting, making some pixels darker and more shadowed than others. We also implement real-time reflections and realistic shadows, which hugely improve visual fidelity and are far more complex to implement using traditional, non-ray traced methods. For post-processing, we implement a blend of three forms of anti-aliasing: multisample, temporal, and spatial. Each of these helps de-noise the scene: without them many pixels are extremely noisy, giving the scene an extremely pixelated and low-quality look, especially when moving the camera. When combined, these techniques significantly reduce framerate, due to anti-aliasing being computationally expensive for real-time 3D graphics. However, the improvement in visual quality for more complex lightning like ambient occlusion makes the performance impact well worth it.

An example of ambient occlusion with realistic shadowing.

High-level architecture

Our project has four core steps in its execution. First, the scene is prepared, with the various light sources and objects being converted from .obj files to arrays of points in preparation for rendering. Then, the ray tracing stage occurs, calling a compute shader invocation on each pixel in the scene. This stage creates rays, intersects them with scene geometry, and simulates the rays bouncing realistically between objects and light sources. It outputs the computed color, normal, and depth value for each pixel into a 2D texture for storage. The post-processing phase, run on a seperate compute shader, performs anti-aliasing on each pixel in the ray traced texture. This post-processing blends pixels from previous frames for each pixel of the current frame, producing a smoother image. The final result of this stage is written back into the same 2D texture. Finally, the last phase simply renders the 2D texture output from the post-processing phase onto a 2D rectangle, producing the final scene as a 3D image.

Functionality

  • Use W A S D, space, and left shift to move the camera around the scene. t
  • Use QEZC to rotate the camera around its current position.
  • Use F to toggle the aspect ratio between 4:3 and 16:9.
  • Use 1, 2, and 3 to toggle between different scenes.
